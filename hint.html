Hint for Solving the DOM XSS Lab

Hint:

This lab has a DOM-based Cross-Site Scripting (XSS) vulnerability. To solve it, consider the following:

  • Look at how the URL parameters are being handled by JavaScript on the page.
  • Try injecting a JavaScript payload into the URL by using a query parameter.
  • The vulnerable function here is document.write(), which is known to cause security risks when used with untrusted input.
  • Craft a payload that can be executed by this function when you control the URL parameters.
  • Example payload to try: ?input=<script>alert('XSS Attack!')</script>

Extra Tip:

Make sure to check the instructions on how to manipulate the URL parameters to inject your script.
